Site Ideas

Two lists from the former Resources page (“Ideas for this Website”) and the former Learning page (“Clear Structure Ideas”), kept as separate sections.

Ideas for this Website

  • Collect Resources and organize
  • Create "incorrect" samples with tutorial
    • Explore JavaScript libraries for guided tours suh as Shepard.js and Intro.js
    • The above libraries allow for defined steps, highlighted elements, and tooltips.
    • I would define each section, write clear instructions, and integrate ARIA guidance into the tour
    • Build my pages like any normal site, then layer the tour library on top to walk users through examples.
  • Create user stories for accessibility users
  • Create website with: forms, headings, images, & buttons
    • Leave out features such as alt text, labels
    • Bad semantics, low contrast, bad headings
    • Run WAVE or Axe on it
    • Practice remediating each issue they flag
  • Create a quiz component
  • Create downloadable checklists
  • Create a glossary
    • Could add tags like a blog, to filter terms by department
  • Data visualization on departments using P.O.U.R.
    • The idea here would be to segment what's most relevant to depts.
  • Site-wide search feature
  • Build templates for documentation (frameworks)

    Examples include:

    • Project management templates
    • Accessibility reports
  • Create checklists to track my knowledge progress
  • Create downloadable checklists for P.O.U.R., by content type, by disability type, etc.
  • Create "Connect A to B" drag lists, such as for elements to P.O.U.R. principles
  • Add a search more to the top of each page to more easily search for information

Clear Structure Ideas

  • Same-page tabs like on Apple's accessibility website
  • Rearrangement of main navigation (add subnavigation)
  • Add a search more to the top of each page to more easily search for information
  • Add a glossary to the site
  • Add a contact page
  • Add a privacy policy
  • Add a terms of service
  • Add a copyright notice
  • Add a site map
  • FAQs
  • On each page, lead with a brief summary so users know what to expect.
  • Use subheadings and collapsible sections if content is long.
  • Breadcrumbs
