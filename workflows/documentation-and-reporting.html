Documentation & Reporting

Notes from the former Practice page: “Documentation,” “Usage of Visual Cues,” and the reporting/template bullets from “Frameworks” (excluding the agile-only items moved to Agile & Sprints).

Documentation

  • Write clear, step-by-step guides
    • Often in HTML, PDF, or internal wikis covering accessibility standards
  • Great for practical demos:
    • PowerPoints (slides)
    • Short videos (e.g., "how to fix a PDF tag")

Frameworks (reports & templates)

Same for guidelines, online resources, progress reports, and documentation

  • Accessibility reports
    • W3C org provides sample accessibility audit templates
  • General Project Management templates
    • Status summaries
    • Key issues
    • Next steps
    • Timelines

Usage of Visual Cues

  • Icons ("Note" like a lightbulb)
  • Inline badges
    • Example: "Warning" icon (like a triangle)—#FF8C00 (dark amber)
    • Example: "Key info" icon (like a star)—#0050A0 (deep blue)
    • Example: "Success"—#008000 (rich green)
  • Use accessible SVG icons
    • Font Awesome (free resource with icons like note, alert, and info symbols)
    • Free CDN (link in HTML)
    • Add Icons directly with classes
    • For accessible emphasis colors, use strong contrast
    • Use recognizable cues
  • Ensure proper color contrast
