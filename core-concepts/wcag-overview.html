W.C.A.G. Overview

Notes from the former Foundations page (four principles) and the former Practice page (“Additional AA Considerations Beyond Code”).

They also include the single “Criteria” bullet from the former Learning page.

The WCAG's Four Principles

  1. Perceivable: like text alternatives for images
  2. Operable: like keyboard navigation
  3. Understandable: clear language and predictable behavior
    • In this principle, content is clear and predictable
    • Forms should have clear instructions and error messages
    • Navigation should be consistent
    • Language should be readable
    • Users shouldn't be surprised by unexpected behavior
    • Ensure instructions are intuitive, interactions are consistent, and that users can easily understand what to do
  4. Robust: compatible with assistive technologies
  • Each principle breaks down into success criteria like color contrast or form labels
  • I'd be expected to audit content against these criteria

Additional AA Considerations Beyond Code

  • Flashing Content Avoid content that flashes more than three times per second to prevent seizures
  • Language of Page Identify the language of the page so assitive tech can adjust accordingly
  • Error Prevention For important data (i.e., legal, financial), users should have the chance to review and correct errors
  • Text Spacing Users should be able to adjust line spacing without losing content or functionality

    To allow text spacing adjustments, you ensure your site is compatible with user stylesheets or browser controls. Avoid fixed height containers or text that can't reflow when line height or spacing is increased.
