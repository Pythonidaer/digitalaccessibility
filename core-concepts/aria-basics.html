ARIA Basics

Notes from the former Foundations page: the four ARIA-related sections grouped on one page.

What is ARIA?

  • Accessible Rich Internet Applications
    • Created by the World Wide Web Consortium (W.3.C.)
    • Used to make dynamic UI (JS-heavy apps) accessible to assistive tech
    • Supports W.C.A.G., mainly Perceivable, Operable, & Robust

What is ARIA For?

  • ARIA helps screen readers and assistive tech understand UI that HTML alone doesn't well
      • Examples:
        • Custom dropdowns
        • Modal dialogs
        • Tabs
        • Sliders
        • Live updates (alerts, notifications)

Without ARIA, screen readers may not know what something is.

With ARIA, you explicitly tell it.

What ARIA Actually Does

  • Roles ‐ Tell what something is
    • role = "button"
    • role = "dialog"
    • role = "navigation"
  • States ‐ Tell current condition
    • aria-expanded = "true"
    • aria-checked = "false"
  • Properties ‐ Give extra meaning
    • aria-label = "Close menu"
    • aria-describedby = "error-message"

What ARIA Satisfies in W.C.A.G. (P.O.U.R.)

  • Perceivable
    • Adds labels/descriptions so users can understand content
    • Example: aria-label on icon buttons
  • Operable
    • Helps keyboard & assistive tech interaction
    • Example: aria-controls on menus
  • Understandable (partially)
    • Can clarify meaning (i.e., labels, relationships)
    • But not the main tool here
  • Robust (BIG One)
    • Ensures compatibility with assistive tech
    • This is where ARIA matters most
