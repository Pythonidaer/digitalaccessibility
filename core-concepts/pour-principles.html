P.O.U.R. Elements Broken Down

This is a non-exhaustive list meant to connect examples to specific W.C.A.G. Principles

  • Perceivable
    • Images (alt text) Ensure all images have text alternatives
    • Semantic Headings Use HTML heading elements (i.e., H1, H2, etc.) to structure content logically
    • Color Contrast Ensure sufficient contrast (AA requires a contrast ratio of at least 4.5:1 for normal text)
    • Non-Text Content Provide alternatives for audio, video, or other media
  • Operable
    • Keyboard Navigation All functionality should be operable via keyboard
    • Focus Indicators Ensure clear focus states when navigating via keyboard
    • Consistent Navigation Repeated elements (like menus) should appear in a consistent order
    • Timing Users should have enouigh time for tasks—avoid timeouts or give options to extend time
  • Understandable
    • Forms with Labels Every form input needs clear, programmatically associated labels
    • Readable Text Content should be clear, with a reading level suitable for a wide audience, or with supplemental aids (like definitions)
    • Predictable Behavior Ensure consistent behavior, avoiding sudden changes
  • Robust
    • ARIA Labels Use ARIA attributes when native HTML isn't sufficient
    • Compatible Code Ensure content can be interpreted by assitive technologies (e.g., sreen readers, browsers, etc.)
