Platform-Specific Tools (for assistive tech)

iOS

  • VoiceOver and test whether all elements are spoken and operable by swiping.
    • Use iOS Accessibility Inspector to check labels, traits, and contrast.

Desktop

  • Keyboard navigation, built-in accessibility tools.
  • Screen readers like:
    • NVDA free & open source
      • Allows you to slow down and speed up speech rate playback
    • JAWS commercial product (requires a license)
      • Has trial periods
      • Offers advanced features
      • More in-depth customization, scripting, & better support in certain enterprise applications
  • In both cases > check if all interactive elements are properly labeled, focusable, and properly ordered
  • Microsoft 365 accessibility features (to run checks and tag content)

Websites (to scan for issues)

  • Lighthouse (Chrome DevTools: checks for Accessibility, SEO, Performance, and more.)
  • Axe
    • Chrome Extension: available in Chrome Inspector, with a solid free tier
    • Runs automated accessibility tests on web pages
      • Flags issues with detailed explanations
  • WAVE
    • Comparable to Axe, but likely has unique features
    • Provides a visual overlay on the page
    • Highlights exactly where issues are

PDFs

  • Acrobat Accessibility Checker
    • Available in the free version
    • More robust and advanced remediation tools in Pro

Video Captioning

  • For accurate video captioning:
    • YouTube & Vimeo indeed offer auto-captions, but you would have to refine them manually
    • Both platforms have built-in caption editors
    • Outside them, common tools are:
      • Rev rev.com
      • Amara amara.org
      • Kapwing kapwing.com
    • You can generate captions, then edit them line by line
  • For business standard software, Adobe Premiere Pro & Camtasia alllow in-depth captioning and styling
  • In practice, you would start with auto-generated captions if available, then correct them with these tools, ensuring:
    • timing
    • accuracy, &
    • formatting align with accessibility needs

How to see traffic and analytics

  • Rely on the institution's analytics tools
    • Typically platforms like Google Analytics (for websites)
    • or built-in tools for video platforms (e.g., YouTube Analytics)
  • To access this data, with the IT or marketing teams—they manage analytics dashboards & can grant you access or share reports
    • Recent vs. most views
    • Preferred report or direct access?
  • You would then filter by traffic metrics—page views, unique users, or watch time.
  • High-traffic items (most visited paged or frequently watched videos) should be prioritize for accessibility.
  • So, step one is ensuring you have the right permissions to view that data.

How to audit thousands of hours of automated captioning videos

  1. Conduct a sampling audit—reciew a representative portion of videos to identiy patterns of compliance or gaps, like missing captions
    • if missing -> how to remediate?
  2. Once patterns are clear, prioritize high-impact or high-visibility first
  3. For content creators, provide clear guidelines:
    1. Always generatee captions—either auto or manual
    2. Mandate review—ensure all captions follow W.C.A.G. standards (e.g., speaker ID, sound cues)
  4. Document a clear workflow—when to caption, who reviews, & final checks before publishing
  5. Train editors to use preferred caption tools, & set periodic reviews to ensure ongoing compliance
